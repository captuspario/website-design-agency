# QA Scorecard

Use this scorecard as an acceptance rubric. A site is ready only when every category is at least passable and no blocking risk is open.

## Scoring model

- `Pass`: acceptable for launch or handover
- `Needs work`: fix before sign-off
- `Blocked`: cannot proceed without upstream change or approval

## Rubric

| Area | Pass looks like |
| --- | --- |
| Homepage clarity | Clear business, offer, audience, and next step within the first screen and immediate scroll. |
| Navigation simplicity | Primary navigation is short, obvious, and free of duplicate or confusing paths. |
| Mobile quality | Key pages read well on mobile, spacing holds up, and CTAs remain easy to use. |
| Visual consistency | Type scale, spacing, components, buttons, and imagery feel coherent across the site. |
| Service-page quality | Each service page explains scope, trust cues, and next action without vague filler. |
| Copy clarity | Copy is plain, specific, and free of invented claims or obvious fluff. |
| CTA clarity | Calls to action are visible, relevant, and consistent with page intent. |
| Accessibility basics | Heading order, colour contrast, image alt decisions, focus states, and link clarity meet practical expectations. |
| Performance basics | No obvious heavy media misuse, broken caching assumptions, or avoidable page bloat. |
| Editor usability | A non-technical owner can update common text and images without breaking structure. |
| Analytics readiness | Baseline measurement approach is defined and core conversion events are not ambiguous. |
| SEO baseline | Titles, meta descriptions, heading structure, internal links, and local intent basics are covered. |
| Legal and privacy baseline | Privacy, consent, contact details, and basic legal pages are present or clearly flagged. |
