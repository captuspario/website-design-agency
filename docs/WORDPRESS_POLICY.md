# WordPress Policy

## Environment

- Clean install only
- Local or staging only
- No live production editing during build

## Theme baseline

- Use a free block theme baseline
- Default to Twenty Twenty-Four unless a different approved free block theme is justified
- Keep theme selection conservative and maintainable

## Build method

- Reusable patterns and components first
- Prefer site-wide style controls over ad hoc page overrides
- Avoid page-builder sprawl
- Keep template and pattern decisions easy to understand later

## Plugin policy

- Use a minimal approved plugin set
- Add plugins only when the need is specific and justified
- Avoid plugin overlap and novelty plugins

## Scope guardrails

- No ecommerce
- No memberships
- No LMS
- No custom app behaviour
- No speculative integrations

## Escalation

Escalate before changing theme baseline, adding unusual plugins, or making a build decision that increases long-term maintenance cost.
