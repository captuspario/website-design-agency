# Issue Taxonomy

Use these issue types consistently in audits, QA passes, and handover notes.

| Type | Use when |
| --- | --- |
| UX | Page flow, clarity, CTA logic, or interaction friction is the problem. |
| IA | Navigation, hierarchy, page model, or information grouping is the problem. |
| Copy | Wording, specificity, message hierarchy, or unsupported claims are the problem. |
| Image | Image quality, relevance, consistency, or source suitability is the problem. |
| SEO | Metadata, heading structure, internal linking, or local search hygiene is the problem. |
| Analytics | Measurement setup, event clarity, or reporting readiness is the problem. |
| Technical | Theme, plugin, rendering, responsiveness, or performance basics are the problem. |
| Accessibility | Contrast, semantics, keyboard access, alt treatment, or practical access is the problem. |
| Handover | Editability, owner understanding, or maintenance readiness is the problem. |
| Approval Needed | Work cannot proceed until a human decision is made. |
| Blocked | Progress is blocked by missing assets, missing facts, or unavailable access. |
| Risk | A non-blocking but meaningful concern should stay visible. |
