# Journey Model

## Purpose

Define a lightweight way to describe how likely audience groups move from first impression to enquiry.

## Required structure

For each key audience intent, capture:

- Audience label
- Primary goal
- Main questions
- Main objections
- Trust cues needed
- Best entry pages
- Best CTA path

## Output standard

Keep the model short. One page of practical journey insight is more useful than a large persona document.

## Use

- Inform page intent and CTA logic
- Prioritise trust elements
- Reduce friction in section order and navigation

## Hard limits

- This is an inference tool, not a claim of user-research certainty.
- Escalate if decisions depend on evidence the project does not have.
