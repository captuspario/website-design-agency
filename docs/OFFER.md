# Offer

## Core offer shape

This company is built for a compact, internal-first website redesign offer with five main phases:

1. Compact audit
2. Redesign blueprint
3. WordPress build
4. QA
5. Handover

## Compact audit

- Review current site structure, messaging, trust cues, and friction points.
- Capture business context, category conventions, and obvious conversion gaps.
- Produce a concise issue list and prioritised recommendations.

## Redesign blueprint

- Clarify audience intent, message hierarchy, page purposes, and CTA logic.
- Propose IA changes, section structures, and reusable pattern choices.
- Define what should be kept, merged, rewritten, removed, or added.

## WordPress build

- Translate approved direction into a local or staging WordPress build.
- Use approved block-theme and plugin constraints.
- Prefer reusable patterns and components over page-by-page improvisation.

## QA

- Run UX, accessibility, technical, SEO, analytics, and handover checks.
- Log issues using the repo taxonomy and move back through approval if needed.

## Handover

- Confirm the site is editable by a non-technical owner.
- Provide concise notes on content upkeep, recurring checks, and known limits.

## Optional care plan

An optional care plan can be offered after launch for light updates, QA rechecks, and baseline maintenance review. It should stay small, operational, and clearly separate from major redesign scope.
