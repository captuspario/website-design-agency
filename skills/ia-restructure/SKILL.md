# IA Restructure

## Purpose

Turn messy page sets and unclear navigation into a simpler information architecture that better matches user intent and business priorities.

## Inputs

- Page inventory
- Business context summary
- Audience and journey notes

## Outputs

- Recommended sitemap
- Navigation model
- Merge, split, delete, and add recommendations

## Steps

1. Review the current page set against business priorities.
2. Define a cleaner sitemap and primary navigation.
3. Recommend which pages should be merged, removed, or newly created.
4. Note the service-page model and supporting page types.

## Done criteria

- The sitemap is simpler than the starting point.
- Navigation logic is easy to explain.
- Page recommendations support later design and build work.

## Escalation

Escalate when IA decisions materially change scope, content load, or stakeholder expectations.
