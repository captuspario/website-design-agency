# WordPress Build Plan

## Purpose

Translate approved strategy and design outputs into a concrete production plan before building starts.

## Inputs

- Approved IA and UX/UI direction
- Theme baseline
- Plugin policy

## Outputs

- Build sequence
- Template and pattern plan
- Plugin decision notes
- Open production risks

## Steps

1. List the pages, templates, and patterns that must be built.
2. Map design decisions to theme and editor capabilities.
3. Identify approved plugins that are actually needed.
4. Note dependencies, blockers, and handoff points.

## Done criteria

- The production sequence is clear.
- Plugin and pattern choices are justified.
- Risks are visible before assembly begins.

## Escalation

Escalate when approved direction cannot be delivered cleanly within the WordPress baseline.
