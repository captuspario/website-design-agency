# UX Structure Design

## Purpose

Define how each important page should flow so the user understands the offer, gains trust, and knows what to do next.

## Inputs

- IA plan
- Messaging framework
- Journey notes

## Outputs

- Page structure notes
- Section order guidance
- CTA placement logic
- Friction reduction notes

## Steps

1. Define the purpose of each priority page.
2. Sequence sections in a way that supports user decisions.
3. Place CTAs where they match readiness and intent.
4. Remove confusing jumps, repeated sections, and dead-end patterns.

## Done criteria

- Each key page has a clear structure.
- CTA logic is intentional rather than generic.
- Section order reflects user needs, not internal wish lists.

## Escalation

Escalate when copy, proof, or IA decisions are still too unstable for page-structure work.
