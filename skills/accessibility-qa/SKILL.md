# Accessibility QA

## Purpose

Catch practical accessibility issues early enough to fix them before handover or launch.

## Inputs

- Built site
- QA scorecard

## Outputs

- Accessibility issue list
- Risk notes
- Recommended fixes or escalations

## Steps

1. Review heading order and semantic structure.
2. Check contrast, focus visibility, and link clarity.
3. Review image alt treatment and obvious keyboard traps.
4. Flag areas that need more formal review if risk appears high.

## Done criteria

- Findings are specific and actionable.
- Risk is stated conservatively, not overclaimed.
- Practical basics are covered across key pages.

## Escalation

Escalate when the project needs specialist accessibility testing beyond baseline review.
