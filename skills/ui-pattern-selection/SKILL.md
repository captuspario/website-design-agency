# UI Pattern Selection

## Purpose

Select a restrained set of reusable patterns that make the site coherent and buildable instead of improvised page by page.

## Inputs

- UX structure notes
- Design direction
- Theme baseline

## Outputs

- Pattern shortlist
- Component usage notes
- Reuse rules

## Steps

1. Review key page structures and recurring content needs.
2. Choose reusable patterns for hero, proof, service, CTA, FAQ, and contact sections where relevant.
3. Check that selected patterns fit the theme and block-editor baseline.
4. Note what should stay standardised across the site.

## Done criteria

- Pattern choices reduce build sprawl.
- The shortlist is realistic for the WordPress setup.
- Reuse rules are clear enough for production.

## Escalation

Escalate when requested visual patterns add unnecessary complexity or conflict with the approved baseline.
