# Page Inventory

## Purpose

Create a simple record of what pages exist, what they do, and whether they should stay, change, merge, or go.

## Inputs

- Existing website
- Sitemap if available

## Outputs

- Page list
- Page type grouping
- Keep, merge, remove, or rewrite notes

## Steps

1. List all current pages and obvious templates.
2. Note each page’s current purpose and likely audience.
3. Mark duplicates, dead ends, and weak service-page coverage.
4. Recommend keep, merge, remove, or rebuild actions.

## Done criteria

- The list is complete enough to support IA planning.
- Every key page has a clear disposition note.
- Duplicate or low-value pages are visible.

## Escalation

Escalate when the current site cannot be accessed fully or when hidden pages may materially affect the IA decision.
