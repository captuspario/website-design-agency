# Audit Report

## Purpose

Convert raw review notes into a concise report that leadership and clients can act on quickly.

## Inputs

- Audit findings
- Issue taxonomy
- Priority notes

## Outputs

- Compact audit report
- Prioritised issues
- Recommended next actions

## Steps

1. Group findings by issue type and severity.
2. Summarise the most important website problems first.
3. Keep recommendations practical and within scope.
4. Call out assumptions, blockers, and approval needs.

## Done criteria

- The report is short and actionable.
- Highest-impact issues are easy to find.
- Findings align with the issue taxonomy.

## Escalation

Escalate when findings are too weak, contradictory, or incomplete to support a useful report.
