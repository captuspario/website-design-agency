# Competitor Benchmarking

## Purpose

Show what the category tends to do well, what it repeats badly, and where the client can be clearer or more credible.

## Inputs

- Competitor URLs
- Category keywords
- Existing website

## Outputs

- Competitor notes
- Category pattern summary
- Weak cliché list
- Differentiation opportunities

## Steps

1. Select a small, relevant competitor set.
2. Review homepage, service pages, and contact paths.
3. Note repeated trust cues, IA patterns, and CTA conventions.
4. Mark weak or generic patterns that reduce trust.
5. Translate observations into practical direction for the project.

## Done criteria

- Findings go beyond surface style comments.
- The output contains actionable implications for content, IA, UX, or UI.
- Copycat recommendations are avoided.

## Escalation

Escalate when no credible benchmark set is available or when category differences are too large for direct comparison.
