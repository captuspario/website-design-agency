# Content Messaging Framework

## Purpose

Turn business and audience insight into a structured messaging direction the rest of the team can build from.

## Inputs

- Business context summary
- Audience summary
- Competitor notes
- Existing content

## Outputs

- Message hierarchy
- Page intent notes
- CTA logic
- Proof requirements

## Steps

1. Define the main promise and supporting messages.
2. Assign clear page-level intent to key pages.
3. Specify the CTA logic for each page type.
4. List the proof, examples, or reassurance needed.
5. Flag unsupported claims that need validation or removal.

## Done criteria

- Messaging is specific enough to direct copy and design.
- CTA logic matches page intent.
- Unsupported claims are not carried forward.

## Escalation

Escalate when core claims, proof, or service details are missing and the messaging would otherwise become speculative.
