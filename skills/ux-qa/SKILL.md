# UX QA

## Purpose

Check whether the site is understandable, easy to use, and consistent across important pages and devices.

## Inputs

- Built site
- Workflow and QA scorecard

## Outputs

- UX issue list
- Severity notes
- Recommended fixes

## Steps

1. Review homepage clarity and page-level intent.
2. Check navigation, CTA logic, and section flow.
3. Review mobile behaviour and responsive readability.
4. Record inconsistent or confusing patterns.

## Done criteria

- Issues are concrete and reproducible.
- Mobile quality is included.
- Findings map back to the QA scorecard.

## Escalation

Escalate when fixing a UX issue would require upstream IA or strategy changes.
