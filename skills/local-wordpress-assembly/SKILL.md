# Local WordPress Assembly

## Purpose

Build the approved website direction in WordPress without drifting from scope, maintainability, or environment rules.

## Inputs

- WordPress build plan
- Approved copy and image direction
- Theme and plugin decisions

## Outputs

- Local or staging WordPress build
- Reusable patterns and templates
- Build notes

## Steps

1. Start from a clean install.
2. Configure the approved theme baseline and minimum plugins.
3. Build reusable templates and patterns first.
4. Assemble pages using approved structure and content direction.
5. Record build notes and known constraints for QA.

## Done criteria

- The build exists only in local or staging.
- Reusable patterns are preferred over ad hoc editing.
- The result is stable enough for QA.

## Escalation

Escalate when requested changes require live production edits, unsupported plugins, or pattern-breaking customisation.
