# Journey Mapping Lite

## Purpose

Show how likely visitors move from first impression to enquiry so page flow and CTA logic can be designed with intent.

## Inputs

- Audience needs summary
- Business context summary
- Existing or proposed page set

## Outputs

- Key audience-intent journeys
- Entry pages and CTA path notes
- Friction and proof needs by journey

## Steps

1. Choose the main audience intents that matter to the website.
2. Define likely entry points and the next useful decisions.
3. Identify friction, proof needs, and CTA timing.
4. Summarise the journey in compact decision-oriented language.

## Done criteria

- The model is concise.
- Each journey points to practical layout or CTA implications.
- Uncertain assumptions are labelled.

## Escalation

Escalate when journey design depends on missing service, audience, or conversion details.
